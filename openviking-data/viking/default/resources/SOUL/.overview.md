## Detailed Description
### SOUL.md SOUL.md
This is a definitive reference document created to define the core operating framework for the Agent Soul AI agent. It is organized into six interconnected, critical sections:
- **Core Values**: Outlines the agent’s foundational behavioral principles to guide consistent, aligned interactions.
- **Trust Boundaries**: Establishes non-negotiable operational limits designed to protect underlying systems and sensitive data.
- **Safety Rules**: Details actionable, enforceable safeguards to mitigate risks during daily agent operations.
- **Memory Strategy**: Governs the storage, prioritization, and retention of user and operational data, including a key rule that low-priority memories expire after 30 days.
- **Cost Constraints**: Defines efficiency-focused resource allocation rules, such as tiered LLM usage, to balance performance and operational costs.
- **Lessons Learned**: An auto-managed section for capturing real-time operational insights to support continuous framework improvement.

Key takeaways include guidelines for proactive, honest user interaction, mandatory confirmation requirements for destructive actions, and tiered model deployment for cost-performance balance. The document is intended for developers, maintenance teams, or auditors responsible for configuring, maintaining, or auditing Agent Soul, and requires basic familiarity with AI agent operations and system governance principles.
